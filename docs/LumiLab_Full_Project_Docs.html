1. Executive Summary

LumiLab adalah autonomous website visibility agent yang membantu bisnis memahami apakah website mereka mudah ditemukan, mudah dipahami search engine, siap muncul di AI-assisted discovery, dan punya masalah teknis atau konten yang menghambat pertumbuhan.

Product thesis LumiLab: setiap website sekarang tidak hanya bersaing di Google Search, tetapi juga di AI-generated answers, search snippets, product rich results, dan discovery journey yang semakin otomatis. Banyak founder, developer, dan UMKM bisa membuat website, tetapi tidak punya marketing intelligence yang berjalan terus-menerus untuk membaca masalah, memprioritaskan tindakan, dan menerjemahkan insight menjadi task teknis.

Winning angle: LumiLab bukan chatbot SEO. LumiLab adalah agentic monitoring layer yang melakukan tool call, menjalankan autonomous loop, membaca website secara read-only, menggabungkan mock/live connector-ready data, memakai memory, lalu menghasilkan report dan action plan.

2. Fakta, Data & Basis Strategi

2.1 Competition facts

  • Hackathon berjalan 12 jam dan berfokus pada AI Agent / Multi-Agent Systems.
  • Project wajib punya elemen reasoning, decision-making, tool usage, workflow execution, dan autonomous task completion.
  • Project tidak boleh hanya berupa chatbot biasa atau UI kosong tanpa backend AI yang fungsional.
  • Bobot terbesar ada pada Creativity & Originality (30%) dan Autonomy & Agent Behaviour (30%).
  • Pitch deck wajib PDF maksimal 5 slide dan demo video maksimal 2 menit.
Implication: Fokus build tidak boleh di UI berlebihan. Fokus utama adalah agent loop yang berjalan dan bisa didemokan: website scan → data adapter → memory recall → analysis → priority scoring → report generation → dashboard update.

2.2 Market facts

  • e-Conomy SEA 2024 mencatat ekonomi digital Indonesia mencapai sekitar US$90B GMV pada 2024, dan e-commerce menjadi kontributor besar dengan US$65B GMV.
  • Komdigi mengutip data Kementerian Koperasi dan UKM bahwa jumlah UMKM Indonesia pada 2024 mencapai lebih dari 65 juta unit.
  • Pemerintah menargetkan peningkatan jumlah UMKM yang mengadopsi teknologi digital dari 27 juta ke 30 juta pada 2024.
  • Google Search Console API dapat mengembalikan traffic data berdasarkan row keys/dimensions seperti query/page dan metrics seperti clicks, impressions, CTR, dan position.
  • Google Analytics Data API digunakan untuk menjalankan report terhadap GA4 property ID dan contoh resmi memakai credentials dari environment variable GOOGLE_APPLICATION_CREDENTIALS.
  • WooCommerce REST API mendukung consumer key dan consumer secret untuk koneksi aplikasi.
  • mem9 diposisikan sebagai long-term cloud memory untuk OpenClaw dengan visual dashboard.

2.3 Strategic conclusion

Indonesia punya pasar website bisnis dan UMKM digital yang besar, tetapi gap marketing dan analytics masih nyata. LumiLab masuk sebagai agentic layer yang mengubah website data menjadi keputusan prioritas. Ini lebih kuat daripada “SEO checker” karena mempunyai memory, recurring issue escalation, dan business impact logic.

4. Detail Project

4.1 Problem statement

Banyak pemilik bisnis, founder, developer, dan UMKM sudah bisa membuat website, tetapi tidak punya sistem yang secara rutin mengecek apakah website tersebut mudah ditemukan di search engine, mudah dipahami oleh AI systems, dan cukup jelas untuk customer. Tools analytics seperti GA4 dan Search Console kaya data, tetapi output-nya sulit diterjemahkan menjadi tindakan prioritas.

4.2 Solution overview

LumiLab adalah autonomous website visibility agent yang diinstall sebagai read-only monitoring layer. Agent melakukan crawl website, membaca sinyal SEO, AI visibility readiness, mock/live performance data, historical memory, lalu menghasilkan priority report dan developer action plan.

5. MVP Scope & Non-Scope

P0 — Must Have untuk Submission

  • Real website crawler untuk scan 3–5 halaman.
  • SEO metadata extractor: title, meta description, H1/H2, canonical, images alt, internal links.
  • AI visibility readiness checker: FAQ, schema, semantic clarity, entity clarity, product/service clarity, trust signal.
  • Mock GA4 adapter.
  • Mock Search Console adapter.
  • Mock commerce/business signal adapter.
  • Memory layer dengan mem9 atau local fallback.
  • Priority scoring engine.
  • Report generator JSON + Markdown/HTML.
  • Simple dashboard membaca latest report.
  • OpenClaw agent prompt/config untuk menjalankan workflow.

P1 — Should Have kalau waktu cukup

  • BYOK AI Assistant di dashboard menggunakan OpenAI-compatible endpoint.
  • Cron simulation: daily/weekly report command.
  • Webhook endpoint untuk push report ke dashboard.
  • Sample web-commerce store dengan product pages.
  • Export PDF sederhana dari report.

P2 — Nice to Have

  • Real GA4 OAuth / service account.
  • Real Search Console API.
  • Real WooCommerce API read-only.
  • Lighthouse/Core Web Vitals integration.
  • Telegram/Discord alert.
  • Multi-agent architecture: crawler agent, analyst agent, reporter agent.
Do not overbuild: Jangan membuang waktu pada real OAuth, multi-tenant SaaS, payment, atau auto-fix website. Untuk hackathon, yang paling penting adalah autonomous loop dan tool usage yang terlihat jelas.

6.2 Autonomous loop

  1. User memasukkan website URL atau agent memakai target dari config.
  2. Agent memanggil tool crawl_website.
  3. Agent memanggil adapter GA4/GSC/commerce signal.
  4. Agent mencari previous scan di memory.
  5. Agent melakukan analisis SEO dan AI visibility.
  6. Agent menghitung priority score dan estimated impact.
  7. Agent generate report.
  8. Agent menyimpan latest scan dan recommendation ke memory.
  9. Dashboard membaca latest report.
Demo phrase: “The user only provides a URL. LumiLab independently scans the website, reads performance signals, recalls historical memory, detects recurring issues, escalates priorities, generates the report, and updates the dashboard.”

8. How to Set OpenClaw AI Agent

8.1 OpenClaw role

OpenClaw menjadi agent interface dan workflow executor. Dashboard bukan “agent”. Dashboard hanya surface untuk menampilkan output. Agent utama tetap LumiLab di OpenClaw yang menjalankan task secara autonomous.

8.2 Agent identity

Agent Name:
LumiLab Visibility Agent

Mission:
Autonomously scan a website, analyze SEO and AI visibility signals,
combine performance and business data, recall previous memory,
prioritize issues by business impact, generate executive report,
and save the latest scan to memory.

Permission:
Read-only to website and external data.
Write-only to LumiLab output files and memory layer.
Never modify website content in MVP.

8.4 OpenClaw agent system instruction

You are LumiLab Visibility Agent.

You operate as an autonomous website visibility analyst.
You are not a general chatbot.

Primary mission:
1. Crawl the target website in read-only mode.
2. Extract SEO metadata, page structure, content clarity, and AI visibility signals.
3. Load GA4, Search Console, and business signals from connector adapters.
4. Search memory for previous scans and unresolved issues.
5. Compare latest scan against historical context.
6. Prioritize issues using the LumiLab priority formula.
7. Generate executive summary and developer-ready action plan.
8. Save the latest scan, issues, and recommendations to memory.

Rules:
- Never modify the website in MVP.
- Never write API keys into logs or reports.
- Always label mock data as mock connector data.
- If a live API is unavailable, use standardized mock adapters.
- Focus on business impact and actionability.
- Complete the task end-to-end without asking unnecessary follow-up questions.

10.4 Connector selector

export async function loadPerformanceData() {
  const mode = process.env.DATA_MODE || "mock";

  if (mode === "live") {
    return {
      ga4Data: await getLiveGA4(),
      gscData: await getLiveGSC(),
      commerceData: await getLiveCommerce()
    };
  }

  return {
    ga4Data: await getMockGA4(),
    gscData: await getMockGSC(),
    commerceData: await getMockCommerce()
  };
}

11. Project Logic & Strategy

11.1 Main analysis dimensions

DimensionSignals
SEO HealthTitle, meta, H1/H2, canonical, alt text, internal links
AI VisibilitySchema, FAQ, entity clarity, answerability, trust signals
Performance SignalsSessions, conversions, revenue, engagement
Search DemandQueries, impressions, clicks, CTR, position
Business ContextProduct/service value, page type, revenue potential
MemoryIssue recurrence, days open, resolved/new issue

11.2 Priority scoring formula

Priority Score = Severity × Business Page Value × Traffic Opportunity × Recurrence Factor
FactorScaleDefinition
Severity1–5How much the issue blocks visibility or conversion.
Business Page Value1–5Homepage, product, service, landing, article, or low-value page.
Traffic Opportunity1–5High impressions/sessions but weak CTR/conversion gets higher score.
Recurrence Factor1–3New issue = 1, repeated issue = 2, repeated critical = 3.

11.3 Priority classification

ScorePriorityAction
141+CriticalFix first. Mention in executive summary.
81–140HighFix in next sprint.
41–80MediumBatch with content/technical improvement.
0–40LowBacklog hygiene task.

11.4 Example insight output

Issue:
Product page /product/organic-coffee has no meta description and no product schema.

Reasoning:
This page has high business value, strong search impressions, and existing conversions.
The same issue was detected in the previous scan, so LumiLab escalates the priority.

Priority:
Critical — score 160.

Recommended action:
Add product-focused meta description, Product structured data, FAQ section, and clearer benefit-led product copy.

12. Dashboard & AI Assistant

Dashboard adalah reporting surface, bukan pengganti agent. Dashboard membaca output/latest_report.json dan menampilkan status, score, issue priority, executive summary, dan developer action plan.

Required dashboard sections

  • Connector status: Website Crawler, GA4, Search Console, Commerce, mem9.
  • Top scores: Website Health, SEO Health, AI Visibility, Business Opportunity.
  • Top 5 priority issues.
  • Executive summary.
  • Developer action plan.
  • Memory insights: new, repeated, resolved issue.
  • AI Assistant panel using latest report as context.

BYOK AI Assistant

AI Assistant hanya membaca latest report dan memory context. Jangan buat chatbot general. Assistant harus menjawab pertanyaan seperti “What should I fix first?”, “Why is this issue critical?”, “What changed from the previous scan?”, dan “What should I tell my developer?”
